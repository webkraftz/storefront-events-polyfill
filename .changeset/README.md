## Workflow for this repository

1. Every PR that changes behavior must include a changeset:

   ```bash
   npm run changeset
   ```

   You'll be prompted to pick a bump type (patch / minor / major) and write a one-line summary.

2. The changeset file is committed alongside your code changes.

3. When merged to `main`, the [release workflow](../.github/workflows/release.yml) either:
   - Opens or updates a "Version Packages" PR if there are unreleased changesets
   - Publishes to npm when the version PR is merged
